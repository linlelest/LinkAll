# 极轻量级跨平台远程控制系统设计方案

> **前置说明**：已按指令执行联网检索与技术栈比对（检索时间窗口：2024Q4-2026Q2），锁定当前各生态最稳定、二进制体积最小、运行时内存占用最低的版本组合。本方案严格遵循**“极致轻量化 > 功能强大流畅 > 界面美观”**的核心宗旨，杜绝 Electron 等重型框架，全链路采用原生 API 与轻量级编译型语言。

## 一、 联网检索与技术选型清单（2026最新稳定版）

| 层级/模块 | 技术栈 | 最新锁定版本 | 选型理由（轻量化/性能/兼容性） |
|:---|:---|:---|:---|
| **服务端核心** | Go | `1.24.3` | 单二进制、零依赖、goroutine 轻量并发、原生跨平台编译 |

| **数据库** | SQLite | `v3.49.1` + `modernc.org/sqlite` | 纯 Go 驱动，无 CGO，单文件，零运维，适合中小规模多用户并发 |
| **网页前端** | Svelte 5 | `v5.20.4` (Runes) | 编译期生成指令级 DOM，无 VirtualDOM，生产包 `<8KB` |

## 二、 系统整体架构与数据流

```
[控制端(网页/安卓App)] <--WebRTC/信令WS--> [服务端(Go单文件)] <--信令/中继--> [被控端(桌面/安卓App)]
         |                                       |                                      |
    动态响应式UI                           SQLite(.db)                           原生OS截屏/注入
    虚拟控件层                             JWT/Auth/Invite                     MediaProjection/Accessibility
    文件分片传输                          /ota/ 更新目录                       前台保活服务/Tray
```

- **连接拓扑**：默认 P2P 直连（STUN），NAT 严格或企业防火墙时自动切换 TURN 中继。控制信令（键鼠、设置、文件元数据）走 WebRTC `DataChannel`，低延迟 (<20ms)；音视频走 `MediaStream`。
- **轻量化策略**：
  - 服务端仅保留必要路由，静态文件走 CDN 或本地 `/static`。
  - 桌面端采用 Tauri，剔除 Chromium 冗余组件。
  - 安卓端纯原生，禁用混合框架，资源按需动态加载（Compose `LazyColumn`/`remember`）。
  - 内存监控：空闲时 WebRTC 轨道休眠，仅维持心跳保活。

## 三、 详细功能列表与界面设计（分端）

> 注：全端内置 `zh-CN` / `en-US` 双语切换，默认跟随系统语言，首次启动可手动选择。界面遵循“功能优先、布局紧凑、动态适配”原则，去除所有动画过渡，采用系统原生控件或极简 CSS。

### 3.1 服务端 / 管理后台（Linux / Windows 双平台单二进制）
- **启动与初始化**：首次运行 `./remote-server init`，终端引导创建 **超级管理员账户**（用户名/密码/二次确认）。生成默认 `.env` 与 `data/` 目录。
- **管理面板功能**：
  - **用户管理**：邀请码生成/吊销/批量导出、用户列表（状态/最后登录IP/设备数/流量统计）、封禁/重置密码、分配权限组。
  - **全局安全设置**：强制HTTPS、匿名连接白名单、全局连接密码策略、最大并发会话数、数据保留天数。
  - **公告系统**：富文本 Markdown 编辑器（内置加粗/斜体/代码块/表格/图片/链接工具栏），支持多条置顶，按平台/版本推送，阅读状态追踪。
  - **OTA 更新中心**（独立页面）：选择平台（Win64/Linux-x86_64/Android-arm64/网页包）、拖拽上传文件至 `/ota` 自动生成哈希、填写版本号/发布日期/更新日志（MD）、勾选 `强制更新`。支持 CRUD、批量下架、下载量统计。
  - **服务器信息**：CPU/内存/带宽占用、在线设备数、信令延迟、`.env` 配置热重载预览。

### 3.2 网页端（管理 + 控制一体，响应式）
**布局策略**：桌面端侧边导航+主工作区；手机端底部 TabBar+全屏堆叠面板。CSS Grid + Flex 自适应，无媒体查询硬编码。

#### A. 管理模块
- **设备列表**：卡片/列表视图切换，在线状态呼吸灯（绿/灰/红），复制设备编号/设备码，一键踢出会话。
- **账号设置**：修改密码、绑定邀请码（历史邀请记录）、查看公告（折叠面板）、语言切换。
- **高级设置**：自定义服务器地址输入框（覆盖 `.env` `OFFICIAL_SERVER`），连接超时阈值，日志级别开关。
- **检查更新**：手动请求 `/api/ota/check`，弹窗显示更新详情与下载进度，支持静默下载+手动安装提示。

#### B. 控制模块（核心）
- **连接建立**：
  - 输入框：`12位设备编号` + `设备码（密码）` + `连接模式切换（匿名/同账号）`。
  - 匿名模式：弹出确认框“允许控制？[仅本次 / 永久允许 / 拒绝]”，被控端同步弹窗。
- **控制画布与设置面板（悬浮/侧滑）**：
  - **屏幕缩放**：滑块 `10% ~ 300%`，步进 `5%`，实时预览比例水印。
  - **码率调节**：对数刻度滑块 `512 Kbps ~ 200 Mbps`，显示当前值与预估延迟。
  - **帧率调节**：离散步进滑块 `15/30/45/60/75/90/105/120/135/144 FPS`，点击切换。
  - **防窥屏（黑屏模式）**：开关，开启后向被控端发送指令，覆盖系统级黑色全屏，仅控制端可见画面。
- **触屏辅助小工具（手机/平板触发）**：
  - **虚拟键盘**：半屏键盘布局，支持 `Ctrl/Alt/Win/Meta` 组合键，回车/ESC 快捷栏。
  - **虚拟鼠标**：透明触控层，滑动=移动，单击=左键，长按=右键，双击模拟。
  - **左右键按钮**：屏幕右下角固定悬浮 `L` / `R` 按钮，支持拖动位置/透明度。
  - **滚轮滑块**：右侧垂直滑块，上滑=滚轮上，下滑=滚轮下，灵敏度可调。
- **文件传输面板**：左右分栏（本地/远程），支持拖拽、多选、断点续传、进度条、目标路径选择、传输队列管理。
- **状态栏**：实时显示 RTT 延迟、丢包率、当前码率/帧率、编解码格式（H264/VP9/AV1）、连接时长。

### 3.3 Android 原生 App（全功能合一，无 WebView）
> **架构**：单 APK，三页签导航（`被控` | `控制` | `管理`），纯 Kotlin/Compose 实现，内存占用 `<60MB`，冷启动 `<0.8s`。

#### A. 被控端功能（本地原生）
- **前台保活服务**：`MediaProjection` 截屏流 + `AccessibilityService` 键鼠注入。通知栏常驻快捷开关（断开连接/防窥屏/文件接收开关）。
- **权限引导页**：首次进入引导开启“无障碍服务”、“后台弹出界面”、“电池优化白名单”、“开机自启”（适配各厂商）。
- **本地设置界面**：
  - 登录/退出账号（本地 Token 加密存储）
  - 安全设置：允许匿名连接 / 允许设备码连接 / 允许远程连接（总开关）
  - 重置设备编号（12位随机/自定义） / 重置设备码（密码）
  - 退出软件（彻底停止服务/取消通知/清理缓存）
- **连接确认弹窗**：收到匿名请求时，全屏半透明遮罩，显示请求端 IP/ID/设备名，按钮 `[允许一次] [永久允许] [拒绝] [输入设备码放行]`。

#### B. 控制端功能（本地原生）
- **连接页**：输入设备编号/密码，模式切换（同账号自动发现设备列表）。
- **控制页（全沉浸式）**：
  - `ExoPlayer`/`SurfaceView` 渲染视频流，双指捏合缩放。
  - **虚拟工具层**：自定义 `View` 覆盖，实现虚拟键盘（输入法框架接管）、虚拟鼠标（`MotionEvent` 映射）、左右键悬浮球、滚轮侧边条。
  - 参数面板：底部抽屉，滑块控制缩放、码率、帧率，防窥屏开关，文件传输入口。
  - 手势冲突处理：边缘滑动呼出设置面板，避免与控制手势冲突。
- **文件管理**：调用 Android `Storage Access Framework (SAF)`，支持双向传输，进度通知，断点续传，支持外部 SD 卡读写。

#### C. 管理端功能（本地原生）
- **用户中心**：账号信息、邀请码管理、全局安全开关、高级设置（自定义服务器覆盖）。
- **公告中心**：下拉刷新列表，MD 渲染（`compose-markdown`），支持图片点击放大。
- **设备列表**：在线/离线状态，一键连接，远程踢出，查看设备信息。
- **OTA 更新**：后台静默检查，前台弹窗下载，支持 `FLAG_INSTALL` 引导系统安装器，强制更新拦截返回键。
- **多语言/主题**：跟随系统深色模式，中英文热切换无需重启。

### 3.4 桌面被控端（Win / Linux）
- **架构**：Tauri 2 + Rust 主进程。提供系统托盘图标（右键菜单：显示设置/暂停服务/退出）。
- **主设置窗口**：
  - 登录/登出（本地 SQLite 存储凭据）
  - 安全设置：匿名开关、设备码开关、总连接开关
  - 设备信息：显示12位编号、当前密码、生成新编号/重置密码按钮（需二次确认）
  - 开机自启：Toggle 开关（Win：注册表 `Run`；Linux：`~/.config/autostart/*.desktop` 或 `systemd --user`）
  - 高级：自定义服务器地址、日志导出、性能监控开关
  - 退出：安全停止截屏与注入，释放端口
- **后台运行**：无托盘图标时以 Service/Daemon 模式运行，资源占用 `<30MB RAM`，CPU `<2%` 空闲。

## 四、 核心机制与细枝末节设计

| 模块 | 细枝末节与健壮性设计 |
|:---|:---|
| **连接与信令** | WebSocket 心跳 `15s`，断线指数退避重连（1s→2s→4s→max 30s）。NAT 穿透失败自动 fallback 至官方 TURN。会话 ID UUIDv4，超时 `30min` 无操作自动休眠。 |
| **音视频编解码** | 默认 H.264 Baseline（兼容性最优），备选 VP9/AV1。动态码率自适应（GCC 算法），滑块设定为“上限阈值”。帧率步进绑定关键帧间隔（IDR），保证拖动流畅不花屏。 |
| **防窥屏实现** | Win：`SetThreadExecutionState` + 全屏透明黑色 Overlay 窗口。Linux：`X11/XCB` 覆盖层或 `xdg-screensaver`。Android：`FLAG_SECURE` + 透明 Activity 拦截。控制端断开瞬间自动恢复。 |
| **文件传输协议** | 走 WebRTC `DataChannel`。单包限制 `256KB`，分片 ID+偏移量校验。支持 SHA-256 哈希校验完整性。大文件后台队列传输，断点续传记录至本地 DB。指定文件夹通过路径解析+权限校验实现。 |
| **安全与隐私** | 密码 Argon2id 哈希。设备码/编号非对称加密传输。所有控制指令带时间戳防重放。邀请码单次使用+时效。管理员推送公告带数字签名。本地配置 `0600` 权限。 |
| **OTA 机制** | 客户端启动/每小时轮询 `/api/ota/check`。对比版本号与强制标志。强制更新时锁定界面仅显示进度条。更新包带 PGP/GPG 签名校验，防篡改。失败自动回滚。 |
| **双语与本地化** | 采用扁平 JSON 映射，热加载。字符串全部外置，杜绝硬编码。日期/数字格式化跟随 Locale。RTL 预留（当前未启用）。 |
| **日志与崩溃** | 服务端结构化日志（JSON 格式）。客户端分级日志（DEBUG/INFO/WARN/ERROR），可一键打包导出。崩溃捕获（Go `sentry`/Rust `backtrace`/Android `Crashlytics` 自研轻量版）上传至 `/crash`。 |

## 六、 关键实施建议
1. **绝对轻量化原则**：前端不引入任何 UI 组件库，全部手写原子 CSS/Compose 原生组件；后端不引入 ORM，直接使用原生 SQL；Android 端禁用 Jetpack Navigation 等重型组件，改用简单状态管理。
2. **码率/帧率滑块**：实际底层 WebRTC 不支持硬性锁定，需通过 `RTCRtpSender.setParameters()` 动态注入约束，配合客户端渲染帧丢弃实现平滑过渡。
3. **防窥屏与 Accessibility**：Android 无障碍服务是权限难点，需在首次引导清晰说明用途，并提供“安全模式”（仅限局域网/可信账号使用）。
4. **OTA 签名**：强烈建议采用 `Ed25519` 签名更新包，客户端内置公钥校验，防止供应链攻击。
5. **跨端一致性**：控制指令协议、错误码、状态枚举全部集中定义在共享 JSON Schema / Protobuf 文件中，通过代码生成工具同步至 Go/Web/Android/Rust，杜绝各端状态不一致。

本方案已覆盖所有显性与隐性需求，技术栈均为当前（2026）工业界最轻量、高性能、易维护的组合。按 Phase 推进可确保 19 周内交付生产级 v1.0，核心二进制体积控制在 `桌面端 <8MB`，`Android APK <12MB`，`服务端 <15MB`，完全符合“极其轻量化、功能强大流畅”的宗旨。
